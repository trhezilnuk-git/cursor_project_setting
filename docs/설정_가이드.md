# Cursor · Python 팀 설정 가이드

이 문서는 **처음 설정하는 분**도 끝까지 따라 하면, 저장소에 공통 Cursor/VS Code/Python 설정이 적용되도록 만든 **단계별 안내**입니다.  
중간에 막히면 [문제 해결](#7-문제-해결-자주-묻는-질문)로 이동해 보세요.

---

## 이 가이드로 할 수 있는 것

- 서비스 레포지토리(또는 새 프로젝트)에 **`.cursor`**, **`.vscode`**, **Ruff / pytest / mypy** 설정을 같은 구조로 넣기
- **Cursor**에서 규칙(룰)과 인덱싱 제외(`.cursorignore`)가 자동으로 적용되게 하기
- 팀원 모두 **저장 시 포맷·린트** 등 동작을 맞추기

**전제:** 팀에서 [cursor_project_setting](https://github.com/trhezilnuk-git/cursor_project_setting) 저장소를 그대로 쓰거나, 포크한 뒤 URL만 바꿔서 씁니다.

---

## 0. 시작하기 전에 확인할 것

아래를 하나씩 확인해 주세요. 모두 해당되면 **1단계**로 넘어가면 됩니다.

| 확인 항목 | 설명 |
|-----------|------|
| Cursor 설치 | [cursor.com](https://cursor.com)에서 설치 후 한 번 실행해 둠 |
| Git 설치 | 터미널에서 `git --version` 이 동작함 |
| Python 설치 | 터미널에서 `python3 --version` 또는 `python --version` 이 동작함 (팀 표준이 3.12면 3.12 권장) |
| 터미널 사용 | macOS/Linux: 터미널 앱, Windows: **PowerShell** 또는 **명령 프롬프트** |

> **Windows 사용자:** 아래 명령에서 `source .venv/bin/activate` 대신 **`.venv\Scripts\activate`** 를 사용합니다.

## 1. 어떤 방식으로 적용할지 고르기

둘 중 **하나만** 선택하면 됩니다.

| 방식 | 이런 분께 추천 |
|------|----------------|
| **A. 새 프로젝트** | 지금 막 만들 서비스 레포가 비어 있거나, 이 템플릿을 기반으로 시작하고 싶음 |
| **B. 기존 프로젝트** | 이미 코드와 `pyproject.toml` 등이 있는 레포에 **설정 파일만** 추가하고 싶음 |

선택했으면 해당 섹션만 **순서대로** 진행하세요.

---

## 방식 A — 새 프로젝트(템플릿으로 시작)

### A-1. 템플릿 복제

터미널을 열고, **새 폴더 이름**을 정합니다. 예: `payment-api`.

```bash
cd ~/Documents
git clone https://github.com/trhezilnuk-git/cursor_project_setting.git payment-api
cd payment-api
```

### A-2. 새 Git 히스토리 시작 (선택)

템플릿의 커밋 기록을 쓰지 않고 새 서비스만의 기록으로 시작하려면:

```bash
rm -rf .git
git init
git add .
git commit -m "Initial commit: service skeleton with team Cursor/Python settings"
```

원격 저장소(예: GitHub의 `payment-api`)를 만든 뒤:

```bash
git remote add origin https://github.com/여러분조직/payment-api.git
git branch -M main
git push -u origin main
```

### A-3. 패키지 이름 바꾸기

1. 폴더 `src/your_package` 의 이름을 **실제 패키지 이름**으로 바꿉니다.  
   - 예: `src/payment_api` (폴더명은 보통 **소문자와 밑줄**)
2. 그 안의 `__init__.py` 는 그대로 두어도 되고, 모듈 설명만 수정해도 됩니다.

### A-4. `pyproject.toml` 수정

에디터로 `pyproject.toml` 을 열고 아래를 **본인 서비스에 맞게** 바꿉니다.

| 바꿀 위치 | 예시 |
|-----------|------|
| `[project]` → `name` | `"payment-api"` |
| `[project]` → `description` | 서비스 한 줄 설명 |
| `[tool.hatch.build.targets.wheel]` → `packages` | `["src/payment_api"]` (`src/` 아래 폴더 경로) |
| `[tool.mypy]` → `packages` | `["payment_api"]` (**import 할 때 쓰는 패키지 이름**, `src/` 제외) |

**팀이 Python 3.12만 쓴다면** 추가로:

- `requires-python = ">=3.12"`
- `[tool.mypy]` 의 `python_version = "3.12"`

### A-5. `AGENTS.md` 수정

`AGENTS.md` 를 열어 **스택 이름, 주요 명령, 팀 규칙**을 적어 두면 Cursor가 맥락을 잘 잡습니다.  
한 줄이라도 “이 레포는 결제 API이고 FastAPI를 쓴다”처럼 구체적으로 적는 것을 권장합니다.

### A-6. Python 가상환경과 패키지 설치

프로젝트 **루트**( `pyproject.toml` 이 있는 폴더)에서:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Windows (PowerShell):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

이어서:

```bash
python -m pip install --upgrade pip
pip install -e ".[dev]"
```

### A-7. 잘 되었는지 검사

같은 터미널(가상환경 켜진 상태)에서:

```bash
ruff check .
ruff format .
pytest
mypy src
```

에러가 없으면 **방식 A는 완료**입니다. **3단계(Cursor에서 할 일)** 로 이동하세요.

---

## 방식 B — 기존 프로젝트에 설정만 합치기

### B-1. 작업 위치 확인

**반드시** 해당 서비스 레포의 **최상위 폴더**(루트)에서 작업합니다.  
이 폴더는 나중에 Cursor에서 **File → Open Folder** 로 여는 그 경로와 같아야 합니다.

### B-2. 템플릿을 임시로 받기

```bash
git clone --depth 1 https://github.com/trhezilnuk-git/cursor_project_setting.git /tmp/cursor_project_setting
```

Windows에서 `/tmp` 가 없다면:

```powershell
git clone --depth 1 https://github.com/trhezilnuk-git/cursor_project_setting.git $env:TEMP\cursor_project_setting
```

아래 복사 명령에서 경로만 `C:\Users\...\AppData\Local\Temp\cursor_project_setting` 처럼 바꿉니다.

### B-3. 파일 복사 (덮어쓰기 주의)

**기존에 `.vscode` 나 `pyproject.toml` 이 있다면** 먼저 백업하세요.

```bash
cd /path/to/여러분의/서비스-레포-루트

cp -R /tmp/cursor_project_setting/.cursor .
cp -R /tmp/cursor_project_setting/.vscode .
cp /tmp/cursor_project_setting/.cursorignore .
cp /tmp/cursor_project_setting/.editorconfig .
cp /tmp/cursor_project_setting/.gitattributes .
cp /tmp/cursor_project_setting/.python-version .
cp /tmp/cursor_project_setting/AGENTS.md .
cp /tmp/cursor_project_setting/.env.example .
```

**`pyproject.toml` 이 이미 있는 경우:**  
위에서 `pyproject.toml` 을 **그대로 덮어쓰지 마세요.**  
템플릿의 `pyproject.toml` 을 다른 이름으로 열어 두고, 아래 블록만 **수동으로 합쳐** 넣습니다.

- `[tool.ruff]` 및 하위 설정
- `[tool.pytest.ini_options]`
- `[tool.mypy]`
- `[project.optional-dependencies]` 의 `dev` 목록 (이미 `dev` 가 있으면 패키지 목록만 병합)

`pyproject.toml` 이 **없던** 레포라면:

```bash
cp /tmp/cursor_project_setting/pyproject.toml .
```

그 후 **방식 A의 A-3 ~ A-5** 처럼 패키지 경로·이름·`AGENTS.md` 를 본인 레포에 맞게 수정합니다.

### B-4. `src/your_package` 가 생긴 경우

템플릿을 통째로 합치지 않았다면 이 폴더는 없을 수 있습니다.  
만약 복사 과정에서 `src/your_package` 가 들어왔다면, **삭제**하거나 **본인 패키지 구조에 맞게 이름 변경** 후 `pyproject.toml` 의 `hatch` / `mypy` 경로를 맞춥니다.

### B-5. 가상환경과 검증

프로젝트 루트에서:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -e ".[dev]"
```

이미 venv 를 쓰고 있다면 **그 venv 를 활성화한 뒤** `pip install -e ".[dev]"` 만 실행해도 됩니다.

```bash
ruff check .
ruff format .
pytest
mypy src
```

(`mypy` 대상 경로가 팀에서 다르면 `mypy .` 등으로 조정합니다.)

문제 없으면 **방식 B** 의 파일 반영과 검증은 끝났습니다. 이어서 **3단계(Cursor에서 할 일)** 로 진행하세요.

## 3. Cursor에서 꼭 할 일

### 3-1. 폴더로 프로젝트 열기

1. Cursor 실행  
2. **File → Open Folder…** (또는 **파일 → 폴더 열기**)  
3. **서비스 레포 루트** 폴더 선택  

> **중요:** 하위 폴더만 열면 `.cursor` / `.vscode` 가 적용되지 않을 수 있습니다. **레포 루트**를 엽니다.

### 3-2. 권장 확장(Extension) 설치

1. 왼쪽 사이드바에서 **Extensions**(네 모양 아이콘) 클릭  
2. 상단 검색창에 `@recommended` 입력  
3. **Workspace Recommendations** 가 보이면 **Install** / **Install All**  

또는 팝업이 뜨면 **Install** 을 누릅니다.

필수에 가까운 항목:

- **Ruff** (`charliermarsh.ruff`)
- **Python** (`ms-python.python`)

### 3-3. Python 인터프리터 선택

1. `Cmd+Shift+P` (macOS) 또는 `Ctrl+Shift+P` (Windows/Linux)  
2. **Python: Select Interpreter** 입력 후 선택  
3. **`.venv` 안의 Python** 을 고릅니다 (경로에 `.venv` 포함)

### 3-4. 동작 확인 (에디터에서)

1. 아무 `*.py` 파일을 열고 공백이나 따옴표를 살짝 바꾼 뒤 **저장**  
2. Ruff 포맷이 자동으로 맞는지 확인  

설정은 `.vscode/settings.json` 에 있습니다.

### 3-5. Cursor 규칙(Rules) 확인

1. Cursor 설정 또는 채팅의 **Rules** 관련 메뉴에서 프로젝트 규칙이 로드되는지 확인  
2. 규칙 내용은 `.cursor/rules/*.mdc` 에 있습니다. 팀 정책이 바뀌면 이 파일들을 PR로 수정합니다.

## 4. Git에 커밋하기

새로 생긴 파일을 커밋합니다.

```bash
git status
git add .cursor .vscode .cursorignore .editorconfig .gitattributes .python-version pyproject.toml AGENTS.md .env.example
git add docs 2>/dev/null || true
git commit -m "chore: add shared Cursor and Python workspace settings"
git push
```

`.gitignore` 에 `.venv/` 가 있는지 확인하세요. **가상환경 폴더는 커밋하지 않습니다.**

---

## 5. 팀원에게 알려줄 최소 안내

팀 채널에는 아래 내용만 전달해도 됩니다.

1. 해당 브랜치를 `git pull`  
2. 레포 **루트**를 Cursor로 열기  
3. `@recommended` 확장 설치  
4. `python3 -m venv .venv` 후 `pip install -e ".[dev]"`  
5. Python 인터프리터를 `.venv` 로 선택  

자세한 설명이 필요하면 이 문서 링크를 함께 보냅니다.

## 7. 문제 해결 (자주 묻는 질문)

### Q. Ruff가 저장 시 포맷을 안 해요.

- Ruff 확장이 설치되어 있는지 확인합니다.  
- **Python: Select Interpreter** 가 `.venv` 인지 확인합니다.  
- 명령 팔레트에서 **Format Document With…** → **Ruff** 를 선택해 봅니다.

### Q. `pip install -e ".[dev]"` 에서 오류가 나요.

- `python --version` 이 `pyproject.toml` 의 `requires-python` 을 만족하는지 확인합니다.  
- `pip install --upgrade pip` 후 다시 시도합니다.

### Q. `mypy` 가 모듈을 못 찾아요.

- `[tool.mypy]` 의 `packages` 와 `mypy_path` 가 실제 `src` 구조와 같은지 확인합니다.  
- 팀이 flat layout 이라면 `pyproject.toml` 의 mypy 설정을 팀 레이아웃에 맞게 바꿉니다.

### Q. Cursor가 `.venv` 나 거대한 폴더를 계속 읽어요.

- `.cursorignore` 에 경로를 추가했는지 확인합니다.  
- Cursor를 한 번 다시 열어 봅니다.

### Q. 조직 GitHub 주소가 달라요.

- 이 저장소를 **포크**하거나 **조직으로 이전**한 뒤, 클론 URL을 조직 기준으로 바꿉니다.  
- 가이드 안의 `trhezilnuk-git` URL은 예시이므로, 팀에서 정한 URL로 통일하면 됩니다.
